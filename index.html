最新项目预告

灯带还能够用一个ESP32-Mini的版本替代，但是对于焊接要求很高，光用烙铁实在没有什么把握，暂时延后。

下一个项目现在确定为：《miniBOT平衡车》——这个项目可以接触一下3D打印和外观设计。专业版键盘的打印居然需要额外收取50块钱的费用。

另外时间非常宝贵，机械键盘类似的项目实在没什么玩头的，直接来点嵌入式控制方向的吧。

顺带一提的是，无人机飞行控制得认真玩的，搞不好以后去实习还要用到。

基于ESP8266开发板的WLED灯带控制

材料准备

  1. ESP8266开发板（NodeMCU）

  2. WBS2812灯带。（本次使用的是大二时候买的一直吃灰的74灯珠LED）

  3. microUSB数据线（数据线和电源线是不一样的，必须区分开来）

  4. wled相关固件，以及ESP8266的固件烧录工具。

操作流程

  1. 给ESP8266烧录进固件，连接灯带，D4连接灯带的数据端口。
  2. 打开手机Wifi，搜索WLED的wifi信号，跟随指引给灯带配网。
  3. 然后等到灯带连接上wifi之后就可以使用，找到它的IP地址，直接访问就可以了。

焊接过程

最后应该是废了……不过得到一些经验——

  1. 需要PCB对应的坐标文件方便焊接，提前准备好
  2. 低温焊锡确实好用，可以固定贴片元器件，粘连了一部分也能够正常的用焊台弄好，不过量需要控制，多了很容易粘连。并且要用之前去涂抹，不然很容易糊
  3. 元器件实现全部按照项目准备好，每次用什么拿什么，电阻和电容弄混了很麻烦。

ESP32平衡车

项目地址：迷你掌上平衡车miniBot - 嘉立创EDA开源硬件平台 (oshwhub.com)

代码解析

  1. 这么看确实像，Arudino的语言和C/C++基本一样，具有面向对象的特性，但是在一些标准库函数上和C更加类似。所谓的库封装本质上就是封装了传感器的对象，调用的时候直接使用类的声明定义方法就可以使用它的方法了。

  2. 主函数流程：开启蓝牙——>电机、编码器、模块初始化——>设定PID参数，计算得到PWM参数带入——>循环：获取角度编码值，用PID计算PWM参数——>蓝牙根据模式数据接收/调试参数。

  3. 蓝牙和传感器都是直接基于MPU6050库和ESP32的蓝牙库上，进行了简单的封装。

    1. 传感器使用的是I2C通信。
    2. 蓝牙基本都封装好了，需要注意的是封装额外为蓝牙的回调函数提供了接口——使用了函数作为变量名称。在初始化的时候开启回调函数，当蓝牙连接、中断的时候启动蓝牙回调函数。不同的事件开启不同的部分。
    3. 蓝牙给定了遥感的XY虚拟坐标，根据坐标调整速度目标，然后PID控制能够自动去实现电机转速和角度的相应控制。

  4. moto调速和编码器读取转速：调速的部分是基于Ledc的库进行封装的。

    1. 预编译中用#define定义了单片机的引脚对应的功能。

    2. ledc库的封装：一个函数就可以，第一个参数选择电机，第二个参数输入PWM（0~1023），负数的时候就是倒转。

      1. 电机驱动模块可以看作是一个MOS管驱动的H桥电路。

      2. DRV8333模块可以驱动两个电机，正向和反转各有两个通道，这两个通道的高低电平逻辑决定了电动机的状态——根据芯片手册中的表格可以知道，两个通道中一个通道输入PWM，另一个置0/1可以让一个电机处于正/反、快速/慢速加减速四种状态。

      3. 所谓的通道，就是信号的通道；信号就是随着时间变化的量。单片机中的通道概念，往往是指一路数字信号的输入或者输出。

    3. 编码器的原理和之前用过的那个旋钮很类似，加上了计数功能就有了速度测量的作用。

      1. 编程中的第一步还是开启4个编码器对应接口的中断，中断对应的中断函数依旧是使用了回调函数的做法来实现。
        1. 回调函数可以看作是软件实现的中断，不过相较于真正的中断，这种回调无法设置优先级并且往往无法抢占真正的中断函数。
      2. 使用了volatile关键词，这个词的作用就是保证在多线程的时候一个变量始终“可见”。这样两个线程同步操作这个变量的时候不会发生冲突。
      3. 编码器的原理：在一个上升沿到达时候读取同侧另外一边的高低电平，判断电机正在加速还是减速。

    4. PID算法实现：定义了一个ArduPID类（移植PID库）。这一部分只能够部分看懂。。。

      1. 具体的算法在实现过程中用了2个闭环控制的PID——不是二重闭环，而是独立的两个PID控制，一个负责速度一个负责方向。分别是角度环和速度环。初始化的时候设定好目标（平衡的参数，角度-1°，速度0）和PID参数，compute可以改变输出的角度和速度。

      2. begin是一个类似于构造函数的一个方法；compute方法是PID的核心。

      3. 角度环的输入是MPU6050的读数，编码器中读出左右轮子的速度信息求和作为输入的参数，误差和。目标的设定根据蓝牙传过来的控制信息进行随时的调整。进行运算之后通过引用机制，值直接返回到变量本体。新的变量计算出pwm来更新给电机的PWM驱动。

      4. 速度结果的基础上制造左右电机的速差转向，使得小车变到特定的角度上去。

      5. 为什么PWM可以看成是一个类？本质上PID的作用就是一个闭环调节的“环节”。既然如此，它必然有一个输入和输出——从流程图上很清晰，输入的是输入信号以及误差信号（误差就是输入和预期目标之间的差值），输出计算后的控制信号，具体怎么调节，就是那三个参数该干的事情了。因此在PID类构造的的时候给出的值就是PID三个常量参数，以及输入、预期目标、输出三个可以被读取和改变的参数。

        本次速度环使用的是两个电机速度的加和作为调参的对象。

在Github上搭建个人博客

安装HEXO

hexo是一个快速、简洁并且高效的博客框架。它的内部基于Markdown解析文章，然后生成静待网页。

更加具体的步骤在文档 | Hexo写得非常清晰了。

具体的安装步骤：

  1. 安装好Git和NodeJS

  2. 使用NodeJS中的Npm进行安装$ npm install -g hexo-cli

  3. 安装Hexo之后使用完成部署

Tips:要注意的一点就是，这个本质上是把静态网页部署到了Github上，但是本体并没有一起上传，因此，不能够通过Github进行随意的下载更改——也对，毕竟那样的话理论上谁都可以修改我的博客了。
